# Docker 目录

本目录存放本项目的容器化运行资产，包括镜像构建文件、编排文件、初始化脚本和与容器运行直接相关的最小说明。

本页遵循“山海工枢”对子目录 `index.md` 的规则：只承载目录说明与推荐阅读顺序，不声明站点导航，不替代 `docs/` 下的正式运维文档。

## 1. 当前收录范围

- `Dockerfile`、`.dockerignore` 等镜像构建文件
- `compose*.yml` 等本地、联调或环境级编排文件
- `entrypoint`、初始化脚本、健康检查脚本等运行时辅助文件
- 与容器运行直接相关的环境变量模板和约束说明

## 2. 推荐阅读顺序

1. 本页：了解本目录职责、边界和放置规则
2. `compose*.yml`：查看服务编排、依赖关系和端口暴露
3. `Dockerfile*`：查看镜像构建策略、基础镜像和启动命令
4. `scripts/`、`init/` 或同类子目录：查看初始化、启动和健康检查细节

## 3. 当前状态

- 当前仓库尚未落正式 Docker 构建与编排文件，本页先作为目录入口。
- 后续如果进入工程实现并新增容器化资产，应优先收敛到本目录，而不是散落在仓库根部。
- 正式部署说明、外部服务依赖和环境策略，仍以 `docs/04-project-development/08-operations-maintenance/` 下的人类文档为准。

## 4. 维护规则

- 本目录只放容器化相关工程资产，不重复存放需求、设计、报价或测试文档。
- 需要被 `docs` 站点收录的正式说明，应写入 `docs/`；本目录保留工程侧最小说明和运行资产。
- 新增、删除或移动 Docker 相关文件后，同步更新本页的收录范围和推荐阅读顺序。
- 如后续增加多环境编排，文件命名优先使用 `compose.<env>.yml`、`Dockerfile.<target>` 这类可辨识命名。
