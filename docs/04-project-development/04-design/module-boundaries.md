## 2. 每个模块的边界定义

### `MOD-001` `gateway-api`

- 目标：作为小程序和后台的统一入口，处理鉴权、会话、幂等入口和接口聚合。
- 负责：
  - Token 校验
  - 请求路由
  - API 兼容层
  - 统一错误格式
- 不负责：
  - 订单真相
  - 派单算法
  - 资金台账计算
- 输入来源：乘客小程序、司机端、小程序后台。
- 输出对象：`platform-core`。
- 拥有的数据：会话缓存、请求幂等键。
- 持久化边界：Redis 中的短期会话和幂等键。
- 状态变化：无业务状态机，只处理协议层状态。

暴露接口：

| 接口 ID | 类型 | 调用方 | 说明 |
|---|---|---|---|
| `API-001` | REST | 小程序 | 用户鉴权与会话 |
| `API-002` | REST | 小程序 | 商城 API 聚合 |
| `API-003` | REST | 小程序 | 出行 API 聚合 |
| `API-004` | REST | 后台 | 运营后台 API 聚合 |

允许依赖：

- `platform-core`
- Redis

禁止耦合：

- 直接访问 PostgreSQL 业务表
- 内嵌商城或调度业务规则

失败与降级策略：

- `platform-core` 不可用时返回标准错误码，不缓存伪业务结果。
- Redis 不可用时禁用重复提交保护并触发告警。

运维责任：

- 监控项：请求量、错误率、鉴权失败率、幂等冲突率
- 告警人：后端值守
- 常见故障：上游服务超时、网关限流误伤

### `MOD-002` `identity-driver`

- 目标：管理统一账户、司机档案、角色与权限。
- 负责：
  - 用户基础资料
  - 司机扩展档案
  - 角色与权限判定
- 不负责：
  - 订单支付
  - 钱包和券
- 输入来源：鉴权请求、后台司机配置、业务侧身份查询。
- 输出对象：商城、出行、后台。
- 拥有的数据：用户表、角色表、司机档案表。
- 持久化边界：`identity` schema。
- 状态变化：司机档案状态、角色启用状态。

暴露接口：

| 接口 ID | 类型 | 调用方 | 说明 |
|---|---|---|---|
| `API-005` | REST/Internal | `gateway-api`, `ops-admin` | 用户与司机信息查询 |
| `API-006` | Internal | `commerce`, `ride-order`, `finance` | 角色与司机能力校验 |

允许依赖：

- PostgreSQL

禁止耦合：

- 直接依赖 `dispatch-engine`

失败与降级策略：

- 司机档案状态异常时，阻断分销和接单能力。

运维责任：

- 监控项：注册成功率、司机资料变更失败率
- 告警人：后端值守
- 常见故障：权限错配、身份状态脏数据

### `MOD-003` `commerce`

- 目标：管理商品、二维码归因、商城订单和专区商品。
- 负责：
  - 商品与 SKU
  - 司机二维码归因
  - 商城订单与司机专区订单
- 不负责：
  - 派单逻辑
  - 钱包台账计算
- 输入来源：小程序下单请求、后台商品配置、支付回调。
- 输出对象：`finance`、`ops-admin`。
- 拥有的数据：商品、归因会话、商城订单、专区订单。
- 持久化边界：`commerce` schema。
- 状态变化：商品上架状态、归因窗口状态、订单状态。

暴露接口：

| 接口 ID | 类型 | 调用方 | 说明 |
|---|---|---|---|
| `API-007` | REST | 小程序 | 商品浏览与下单 |
| `API-008` | Internal | `finance` | 支付成功后的订单与归因事实 |

允许依赖：

- `identity-driver`

禁止耦合：

- 直接调用派单服务
- 直接操作钱包余额

失败与降级策略：

- 支付未确认时，订单保持待支付，不触发返佣。

运维责任：

- 监控项：下单成功率、支付成功率、归因命中率
- 告警人：商城业务值守
- 常见故障：归因失效、订单状态未推进

### `MOD-004` `ride-order`

- 目标：承接叫车、行程订单状态机和调度结果落库。
- 负责：
  - 叫车请求与行程订单创建
  - 行程状态流转
  - 调度结果映射到订单
- 不负责：
  - 具体派单算法
  - 钱包返佣计算
- 输入来源：小程序叫车请求、`dispatch-engine` 回调、支付回调。
- 输出对象：`dispatch-engine`、`finance`、`ops-admin`。
- 拥有的数据：行程订单、派单请求记录、订单状态轨迹。
- 持久化边界：`ride` schema。
- 状态变化：待派单、派单中、已接单、进行中、已完成、已支付、已取消。

暴露接口：

| 接口 ID | 类型 | 调用方 | 说明 |
|---|---|---|---|
| `API-009` | REST | 小程序 | 叫车、订单查询、取消 |
| `API-010` | Internal | `dispatch-engine` | 创建派单任务、接受派单反馈 |

允许依赖：

- `identity-driver`
- `dispatch-engine`

禁止耦合：

- 在本模块内部重复保存司机在线短态

失败与降级策略：

- 调度服务不可用时，把订单置为派单失败待处理，不回滚订单创建。

运维责任：

- 监控项：叫车成功率、派单成功率、取消率
- 告警人：出行业务值守
- 常见故障：订单状态滞留、回调重入

### `MOD-005` `finance`

- 目标：统一管理佣金、站内钱包、返现券和退款冲抵。
- 负责：
  - 佣金规则计算
  - 钱包账户与台账
  - 返现券发放与核销
  - 退款逆向分录
- 不负责：
  - 支付渠道本身
  - 商品履约
- 输入来源：商城和出行的支付成功/退款成功事件、司机专区消费请求。
- 输出对象：司机收益视图、后台对账视图。
- 拥有的数据：佣金台账、钱包账户、钱包分录、券发放和核销记录。
- 持久化边界：`finance` schema。
- 状态变化：待入账、已入账、已冲销、已核销。

暴露接口：

| 接口 ID | 类型 | 调用方 | 说明 |
|---|---|---|---|
| `API-011` | Internal | `commerce`, `ride-order` | 支付后返佣与退款冲抵 |
| `API-012` | REST | 小程序/后台 | 收益、余额、券、台账查询 |

允许依赖：

- `commerce`
- `ride-order`

禁止耦合：

- 直接写入调度短态

失败与降级策略：

- 分录失败时标记为待重试，不伪造已完成返佣。

运维责任：

- 监控项：入账延迟、冲销失败率、余额异常数
- 告警人：财务/技术值守
- 常见故障：重复入账、逆向分录缺失

### `MOD-006` `dispatch-engine`

- 目标：管理司机在线态、候选筛选、派单和超时重派。
- 负责：
  - 司机在线/离线状态
  - 地理位置索引
  - 候选司机计算
  - 超时重派
- 不负责：
  - 最终订单真相
  - 佣金或支付逻辑
- 输入来源：司机状态上报、`ride-order` 派单请求。
- 输出对象：`ride-order`。
- 拥有的数据：在线态缓存、派单批次与司机响应记录。
- 持久化边界：Redis + 必要审计回传到主业务系统。
- 状态变化：司机在线状态、派单批次状态。

暴露接口：

| 接口 ID | 类型 | 调用方 | 说明 |
|---|---|---|---|
| `API-013` | Internal | `ride-order` | 创建派单请求 |
| `API-014` | Internal | 司机端/`gateway-api` | 在线状态上报、接单反馈 |

允许依赖：

- Redis
- `ride-order` 内部 API

禁止耦合：

- 直接访问主业务数据库订单表作为实时状态缓存

失败与降级策略：

- Redis 异常时拒绝新派单并回传主业务系统，避免派单失真。

运维责任：

- 监控项：在线司机数、匹配耗时、重派次数
- 告警人：调度后端值守
- 常见故障：在线态失真、派单超时堆积

### `MOD-007` `ops-admin`

- 目标：为运营、财务和支持提供配置和查询能力。
- 负责：
  - 城市配置
  - 商品与规则配置
  - 订单、台账、司机查询
- 不负责：
  - 核心业务规则执行
  - 直接写入资金台账
- 输入来源：后台人员操作。
- 输出对象：配置变更、查询结果。
- 拥有的数据：部分 `ops` schema 配置数据。
- 持久化边界：`ops` schema。
- 状态变化：规则启停、配置版本。

暴露接口：

| 接口 ID | 类型 | 调用方 | 说明 |
|---|---|---|---|
| `API-015` | REST | 运营后台 | 配置与查询接口 |

允许依赖：

- `identity-driver`
- `commerce`
- `ride-order`
- `finance`

禁止耦合：

- 直接操作派单引擎内部缓存

失败与降级策略：

- 配置失败不得覆盖旧规则，采用版本化发布。

运维责任：

- 监控项：配置发布成功率、后台错误率
- 告警人：运营支持 / 后端值守
- 常见故障：规则误发布、查询超时

### `MOD-008` `driver-access`

- 目标：负责司机入驻、资质审核、能力启停和设备绑定。
- 负责：
  - 司机资料提交
  - 资质审核记录
  - 接单/分销/消费能力启停
  - 司机重端设备绑定
- 不负责：
  - 派单计算
  - 收益结算
- 输入来源：司机轻端、运营后台。
- 输出对象：`identity-driver`、`dispatch-engine`、`ops-admin`。
- 拥有的数据：司机审核单、设备绑定记录、能力状态。
- 持久化边界：`identity` / `ops` schema。
- 状态变化：待审核、已通过、已驳回、已暂停。

暴露接口：

| 接口 ID | 类型 | 调用方 | 说明 |
|---|---|---|---|
| `API-016` | REST | 司机轻端/后台 | 司机资料提交、审核、能力启停 |

允许依赖：

- `identity-driver`
- `ops-admin`

禁止耦合：

- 直接参与派单算法

失败与降级策略：

- 审核状态异常时，默认关闭接单能力。

运维责任：

- 监控项：审核积压量、审核通过率、能力启停成功率
- 告警人：运营支持 / 后端值守
- 常见故障：审核状态错乱、设备绑定失败

## 3. 跨模块协作规则

- 调用方向：
  - 外部请求只进入 `gateway-api`
  - `platform-core` 子域之间通过内部服务接口协作
  - `ride-order` 可以同步调用 `dispatch-engine`
- 共享数据规则：
  - 订单与台账以 PostgreSQL 为最终事实
  - 在线态和幂等键允许驻留 Redis
- 事务边界：
  - 单个服务内以本地事务为主
  - 跨服务以状态补偿和幂等保证一致性
- 版本兼容约束：
  - 外部 API 使用版本前缀
  - 内部接口以契约文档为准，变更需先更新文档
