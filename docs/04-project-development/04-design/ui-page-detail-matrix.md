# 页面级详细设计矩阵

**文档状态：** 草稿  
**负责人：** AI 软件工厂  
**主要读者：** 产品 | 交互设计师 | 视觉设计师 | 前端  
**上游输入：** UX/UI 与交互设计 | 端规格文档 | OpenAPI 契约  
**下游输出：** Figma/即时设计/墨刀/ MasterGo 页面稿 | 组件规范 | 前端切图与联调说明  
**最后更新：** 2026-03-29  

## 1. 使用说明

- 本文档服务“页面落地设计”，不是再次讨论产品范围。
- 设计软件出图时，先读取共享设计要素，再按页面矩阵执行。
- 每个页面都必须同时输出：
  - 默认态
  - 空态
  - 加载态
  - 异常态
  - 禁用态或无权限态

## 2. 共享设计要素

### 2.1 尺寸与栅格

| 端 | 设计宽度 | 栅格 | 最小间距 | 卡片圆角 | 说明 |
|---|---|---|---|---|---|
| 乘客小程序 | `375px` | 4 栅格逻辑分区 | `8px` | `16px` | 轻量卡片风格 |
| 司机轻端小程序 | `375px` | 4 栅格逻辑分区 | `8px` | `16px` | 工作台区块化 |
| 司机重端 App | `392px` Android 基准 | 4 栅格逻辑分区 | `8dp` | `14dp` | 大按钮和底部吸附操作区 |
| 运营 Web | `1440px` | 12 栅格 | `12px` | `12px` | 表格与侧栏抽屉优先 |

### 2.2 颜色与状态

| 状态 | 乘客/司机轻端 | 司机重端 | 运营 Web |
|---|---|---|---|
| 主操作 | 深青绿 / 深绿 | 青绿色 | 深蓝 |
| 强提醒 | 橙色 | 橙色 | 琥珀色 |
| 风险/驳回 | 玫红/红 | 玫红/红 | 玫红/红 |
| 成功完成 | 深绿 | 青绿 | 深绿 |
| 禁用 | 灰蓝 | 深灰 | 灰蓝 |

### 2.3 组件清单

| 组件类别 | 组件名 | 使用端 | 说明 |
|---|---|---|---|
| 导航 | 顶部导航栏 | 全端 | 统一标题、返回、城市/筛选入口 |
| 信息摘要 | KPI 卡片 | 司机轻端、运营 Web | 展示收益、订单量、异常量 |
| 业务进度 | 时间线 | 乘客端、司机重端、运营 Web | 展示订单/工单/台账状态流 |
| 行动组件 | 主 CTA 按钮 | 全端 | 高优先级操作；重端要大按钮 |
| 状态标签 | Status Chip | 全端 | 用统一状态色体系 |
| 数据组件 | 表格与筛选栏 | 运营 Web | 支持批量操作、导出、固定列 |
| 反馈组件 | 空态卡、错误提示条、骨架屏 | 全端 | 不允许纯空白页面 |

### 2.4 文案规则

- CTA 文案统一用动词开头，如“立即叫车”“提交审核”“人工指派”。
- 状态文案必须是业务状态，不使用“处理中请稍后”一类模糊文案替代。
- 收益文案必须带条件说明，如“支付成功后入账”。
- 高风险动作弹窗必须展示动作影响、操作原因输入框和不可逆提示。

## 7. 设计交付要求

- 每个页面至少输出 1 版桌面稿或移动稿主视图。
- 所有表单页必须额外输出字段校验态。
- 司机重端和后台高风险页必须额外输出确认弹窗稿。
- 所有列表页必须输出空态与筛选后无结果态。
