# 设计软件提示词包

**项目名称：** 千乘坊（ride-loop）  
**文档状态：** 草稿  
**负责人：** AI 软件工厂  
**主要读者：** 产品 | 交互设计师 | 视觉设计师  
**上游输入：** UX/UI 与交互设计 | 页面级详细设计矩阵 | 端规格文档  
**下游输出：** Figma/即时设计/墨刀/ MasterGo AI 生成稿 | 线框稿 | 高保真稿  
**最后更新：** 2026-03-29  

## 1. 使用方式

- 第一步：把“端级主提示词”粘贴给设计软件，先生成整端视觉系统。
- 第二步：再粘贴对应“页面级提示词”，生成单页布局。
- 第三步：用“审稿提示词”让设计软件回查缺失状态、组件和交互。
- 第四步：用 [页面级详细设计矩阵](./ui-page-detail-matrix.md) 对照补字段和状态。

## 2. 通用提示词模板

```text
你是资深移动端/后台产品设计师，请为“千乘坊（ride-loop）”输出可直接开发落地的页面设计。

项目背景：
这是一个“出租车司机分销 + 本地生活商城 + 网约车派单”的 O2O 闭环系统。
武汉单城首发，司机是统一账户，同时具备接单司机、分销商、司机专区消费者三种能力。
返现和返现券只允许站内循环，不支持提现。

输出要求：
1. 输出完整页面结构，不要只给概念图。
2. 明确导航、布局块、组件、字段、按钮、状态标签、空态、异常态、禁用态。
3. 页面必须符合中国用户习惯，信息层级清楚，文案直接。
4. 不要使用 AI 常见的紫色渐变风格，不要做花哨的大面积装饰。
5. 保持可开发落地，优先输出组件化设计。

请按以下格式输出：
- 页面目标
- 页面布局
- 组件清单
- 字段清单
- 关键交互
- 空态/异常态
- 视觉风格说明
```

## 3. 端级主提示词

### 3.1 乘客微信小程序主提示词

```text
请为“千乘坊”设计乘客微信小程序的整体视觉系统和核心页面风格。

设计目标：
- 乘客扫码后直接看到优惠和两个主入口：去购物、去叫车。
- 风格是“城市信任感 + 本地生活优惠”，不是传统重商城首页。
- 页面要轻量、清晰、可信，不展示分销佣金逻辑。

视觉方向：
- 主色：深青绿 #0F766E
- 强 CTA：橙色 #F97316
- 背景：浅绿白 #ECFDF5
- 字体：中文用 HarmonyOS Sans SC，数字强调用 DIN Alternate 风格
- 风格关键词：轻量、可信、优惠清晰、城市服务感

需要你先输出：
- 设计系统页
- 首页
- 商城列表
- 商品详情
- 叫车页
- 订单追踪页

交互要求：
- 首页主 CTA 不超过 2 个
- 商品卡必须显示履约方式
- 叫车页要突出预估价和优惠抵扣
- 订单页要有清晰时间线

请输出可直接用于高保真设计的结构和组件定义。
```

### 3.2 司机轻端微信小程序主提示词

```text
请为“千乘坊”设计司机轻端微信小程序的整体视觉系统和核心页面风格。

产品定位：
这是司机的经营工作台，不是接单工具。
司机在这里完成分销、看收益、司机专区复购、查看工单。

视觉方向：
- 主色：深绿 #047857
- 辅助色：琥珀色 #F59E0B，浅绿背景 #F7FEE7
- 字体：中文用 Alibaba PuHuiTi 3.0，数字用 DIN Alternate 风格
- 风格关键词：工具感、经营感、收益感、区块化

重点页面：
- 工作台首页
- 入驻申请
- 分销中心
- 收益中心
- 司机专区首页
- 司机专区结算页
- 工单中心

关键要求：
- 今日收益、待入账、可用余额必须优先显示
- 司机专区商品必须显示履约方式：到店核销、到店自提、配送
- 负余额提示要严肃但不能像系统报错
- 不出现接单在线开关

请输出可以直接落地图稿的页面系统、组件命名和页面结构。
```

### 3.3 司机重端 App 主提示词

```text
请为“千乘坊”设计司机重端 App 的整体视觉系统和核心履约页面。

产品定位：
这是 Android 优先的司机接单与行程履约工具，强调夜间可读、低分心、高可靠。

视觉方向：
- 背景：深石墨 #0B1220
- 主操作色：青绿 #14B8A6
- 强提醒：橙色 #F97316
- 危险色：玫红 #E11D48
- 字体：中文用 Source Han Sans SC，数字强调用 DIN Alternate 风格
- 风格关键词：车载工具、功能主义、驾驶安全、状态驱动

重点页面：
- 登录与设备绑定
- 权限引导
- 接单工作台
- 派单弹层
- 当前行程页
- 行程异常页
- 申诉页

关键要求：
- 单屏单任务，不要做复杂花哨布局
- 派单弹层必须一眼看到倒计时、上车点、预估收入、接拒按钮
- 当前行程页底部按钮必须大，防误触
- 高风险动作支持二次确认或长按

请输出适合 Android 落地开发的页面结构、组件和状态。
```

### 3.4 运营 Web 主提示词

```text
请为“千乘坊”设计运营 Web 控制台的整体视觉系统和关键后台页面。

产品定位：
这是武汉单城运营控制台，服务城市运营、调度、客服、财务、风控、管理员。
重点不是大屏炫技，而是高信息密度、强筛选、强审计、强权限提示。

视觉方向：
- 主色：深蓝 #1E3A8A
- 辅助色：琥珀 #F59E0B
- 背景：浅灰蓝 #E2E8F0 / 白色卡片
- 字体：IBM Plex Sans + JetBrains Mono 风格数字
- 风格关键词：控制台、数据密度、审计感、专业后台

重点页面：
- 首页总览
- 司机审核列表
- 商品管理
- 派单监控
- 台账与余额
- 退款与售后
- 工单中心
- RBAC 设置

关键要求：
- 高风险按钮必须有动作级权限表现
- 列表和抽屉是主形态
- 审核、退款、权限变更弹窗必须带原因输入
- KPI 卡不要做 BI 大屏效果

请输出可直接给前端和设计师使用的后台页面系统。
```

## 4. 页面级高优先提示词

### 4.1 乘客端扫码落地页

```text
请设计“千乘坊”乘客微信小程序的扫码落地页。

页面目标：
- 让乘客在 3 秒内理解自己拿到了优惠。
- 引导用户进入两个主路径：去购物、去叫车。

页面结构：
- 顶部：城市、定位、关闭/返回
- 中部：司机专属优惠卡、优惠规则入口、司机推荐说明
- 下部：双主 CTA、推荐商品 2-4 个、信任说明

字段要求：
- 优惠金额/折扣
- 司机推荐标识
- 归因有效时间提示
- 推荐商品标题、价格、履约方式

状态要求：
- 归因有效
- 归因已失效但页面可继续浏览
- 无推荐商品
- 网络异常

视觉要求：
- 轻量可信，不做活动会场风
- 主 CTA 清晰，避免 3 个以上大按钮
```

### 4.2 乘客端叫车页

```text
请设计“千乘坊”乘客微信小程序的叫车页。

页面目标：
- 让乘客完成起终点输入、查看预估价、看到优惠抵扣、立即叫车。

页面结构：
- 顶部地址输入区
- 中部预估价卡 + 优惠卡
- 常用地址区
- 底部固定主 CTA

字段要求：
- 起点/终点
- 预估价
- 优惠抵扣说明
- 服务范围提示

状态要求：
- 地址未填写
- 超出武汉范围
- 无可用优惠
- 叫车中

请输出移动端高保真页面结构和交互说明。
```

### 4.3 司机轻端工作台首页

```text
请设计“千乘坊”司机轻端微信小程序的工作台首页。

页面目标：
- 司机一进入就看到收益、分销、司机专区和待办。

页面结构：
- 顶部欢迎区 + 能力状态标签
- 第一屏：收益卡片组（今日收益、待入账、可用余额、临期券）
- 第二屏：分销区块
- 第三屏：司机专区区块
- 第四屏：待办与消息

字段要求：
- 今日收益
- 待入账
- 可用余额
- 负余额提示
- 今日扫码人数
- 今日成交人数
- 司机专区快捷入口

状态要求：
- 审核通过
- 审核中
- 能力暂停
- 无收益
- 负余额

风格要求：
- 数字优先
- 区块清晰
- 工具感强于运营感
```

### 4.4 司机轻端司机专区结算页

```text
请设计“千乘坊”司机轻端微信小程序的司机专区结算页。

页面目标：
- 让司机快速完成券选择、余额抵扣和履约信息填写。

页面结构：
- 商品摘要卡
- 履约方式卡
- 门店/地址表单
- 券选择区
- 余额抵扣区
- 金额汇总区
- 底部提交按钮

关键要求：
- 到店核销、到店自提、配送三种履约方式要展示不同表单
- 券与余额抵扣逻辑清晰
- 提交前显示应付金额与抵扣明细
- 余额不足、券失效、库存不足需要独立状态
```

### 4.5 司机重端接单工作台

```text
请设计“千乘坊”司机重端 App 的接单工作台。

页面目标：
- 司机能清楚知道当前是否可上线、是否有在途订单、设备和权限是否正常。

- 顶部司机状态区
- 中部上线开关大卡
```

```text
请设计“千乘坊”司机重端 App 的当前行程页。

- 支持司机按顺序完成到达、开始行程、结束行程。

页面结构：
- 当前状态头部
- 地图区
- 起终点摘要
```

### 4.7 运营 Web 首页总览

### 4.8 运营 Web 派单监控页

```text
请设计“千乘坊”运营 Web 的派单监控页。

- 让调度运营快速发现派单失败、长时间无人接单、区域异常，并进入人工干预。

- 支持人工指派、取消、区域停运三类动作入口
```

## 5. 审稿提示词

```text
请作为设计评审人，检查这张页面稿是否满足以下要求：

1. 是否完整覆盖默认态、空态、加载态、异常态、禁用态。
2. 是否所有关键字段都能在页面上找到明确位置。
3. 是否主 CTA 明确且不互相竞争。
4. 是否状态颜色和状态标签表达一致。
5. 是否存在无法开发实现的装饰性设计。
6. 是否高风险动作缺少确认弹窗、权限提示或原因输入。
7. 是否存在多余的营销元素干扰主任务。

请逐条指出缺失项，并给出修正建议。
```
